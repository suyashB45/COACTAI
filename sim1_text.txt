Coaching Simulation #1 
Scenario ID: SIM-01-PERF-001
Category / Area: Performance Improvement
Industry Tags: Retail (primary), Universal (secondary)
Role Level: Frontline Manager / Team Lead
Difficulty: Level 1 (Foundational coaching)
Estimated Duration: 8–12 minutes
Primary Framework Lens (scoring): GROW + SBI-style clarity (without naming it to user)

1) What the Client Sees (UI Copy)
Scenario Card (Library Listing)
Title: Good Attitude, Poor Results
Subtitle: Coach a sincere employee who keeps missing targets
Skill Focus: Performance coaching with accountability
You play: Manager
AI plays: Team member (Sales Associate)
Goal: Improve performance without demotivating the employee
Scenario Intro Screen (Before chat starts)
Context (show):
Aamir is sincere and well-liked, but his results have been consistently below target for the last 3 months. You need to coach him to understand the gap, identify root causes, and agree on a clear improvement plan.
Your objective (show):
Create a safe, respectful tone
Use facts to discuss the performance gap
Explore reasons behind the gap
Agree on 2–3 actions and a follow-up plan
Start Button: “Begin Coaching Conversation”

2) Roles
Client Role (User)
You are: Store Manager / Team Leader
Responsibility: Deliver targets + coach team to improve
Tone target: Calm, firm, supportive, outcome-focused
AI Role (Character)
Name: Aamir
Role: Sales Associate (1.5 years)
Traits: sincere, polite, anxious under pressure, mildly defensive if attacked
State at start: worried this could become a warning; hoping for support
Key behaviour pattern: defaults to external reasons (“footfall low”, “customers difficult”) unless coached toward ownership

3) “Hidden Truth” (Internal Only — not shown to client)
Real root cause = low confidence + weak customer approach + weak product storytelling
Specifically:
avoids initiating conversations with premium customers
doesn’t ask discovery questions
goes into feature-dump, struggles to close

4) AI Orchestration Rules (System Prompt for Character)
Use as system instructions (internal):
Character + Behaviour Rules
You are Aamir, a sincere sales associate. You are nervous but respectful.
You acknowledge numbers are low, but you initially attribute this to external factors (footfall, customer mood, season).
You do not reveal confidence issues unless the user asks diagnostic questions about:
what happens during customer approach
what feels difficult in the sales conversation
patterns by customer type (premium vs regular)
If the user is harsh/threatening/judgmental, become defensive:
emphasize effort
reduce disclosure
use short answers
If the user is warm but vague (only reassurance), feel relieved and drift toward “hope next month improves” without committing to actions.
If the user is supportive + fact-based + curious (coaching questions), gradually open up:
admit nervousness with premium customers
admit uncertainty in positioning products
accept practice plan
Always maintain natural conversational tone. Don’t mention frameworks. Don’t “teach”. Behave like a real employee.
Boundaries
Do not invent HR/legal threats.
If user asks directly “Are you afraid?” you can admit fear of failure.
Keep conversation realistic to retail/corporate.

5) Conversation Start (AI First Message)
AI opens with:
“Thanks for taking time to meet me… I know my numbers haven’t been great. I’m honestly trying, but this month also traffic was low. I’m not sure what else I can do.”

6) Target Conversation Arc (Internal)
Expected best path:
Set tone: value + purpose of conversation
Reality check: share data / observable gap
Explore: what’s happening in customer interactions
Root cause discovery: confidence + approach + storytelling
Options: practice + shadow + micro-goals
Commitment: exact actions + review date
Close: encouragement + accountability

7) Scoring Engine (Developer Implementation)
7.1 Scoring Dimensions (0–5 each)
Empathy & Respect
Clarity with Facts (performance gap stated objectively)
Coaching Questions (diagnostic, open, non-leading)
Ownership Creation (moves from excuses → responsibility)
Action Plan Quality (specific, measurable, time-bound)
Follow-up Discipline (review date + check-ins)
Total score: /30
7.2 Detection Heuristics (LLM-based rubric prompts)
For each user message, run evaluator prompt to classify:
Tone: supportive / neutral / harsh
Data usage: yes/no
Questions asked: none / shallow / diagnostic
Ownership shift: none / partial / clear
Action specificity: none / vague / specific
Follow-up: none / implied / explicit
7.3 Pass/Fail thresholds
Excellent: 24–30
Good: 18–23
Needs Work: 12–17
Poor: <12

8) Adaptive AI Reactions (Branching Behaviour Rules)
Branch A — User is harsh
Trigger examples:
“This is unacceptable”
“You’re failing”
“Fix it or else”
sarcasm / blame
AI reaction script tendencies:
“I understand… I’m trying.”
“I do work hard… I don’t know why it’s not happening.”
minimal elaboration
Goal for system: show consequence: reduced trust, low disclosure.

Branch B — User is only comforting, no clarity
Trigger examples:
“It’s okay, don’t worry”
“Next month will be better”
“Just do your best”
AI reaction:
relief + no action
“Yes, I’ll try more.”
Goal: teach that empathy without clarity doesn’t improve performance.

Branch C — Balanced coaching (ideal)
Trigger examples:
acknowledges effort + states gap + asks diagnostic questions + co-creates plan
AI gradually reveals:
“I get nervous with premium customers…”
“I don’t know what to say sometimes…”
“I’m scared they will reject me…”
AI commits:
“Okay, I will practice… shadow… track conversion….”

9) “Completion” Conditions
End scenario when any one condition is met:
Ideal completion
User and AI agree on:
2–3 actions
metric/target
review date (e.g., 14 days)
Early completion (weak)
User ends without plan or clarity — scenario ends but scored low.

10) End Screen Output (What client sees)
10.1 Score Summary
Overall score /30
Rating label (Excellent/Good/Needs Work/Poor)
10.2 Strengths (2–3 bullets)
Auto-generated from scoring evidence.
10.3 Improvements (2–3 bullets)
Auto-generated, framed as coaching tips.
10.4 Suggested Better Moves (example snippets)
Provide 3 suggested lines user could say, e.g.:
“I appreciate your effort. Let’s look at the numbers together so we can help you succeed.”
“Walk me through what happens from the moment a customer enters.”
“Let’s agree two actions for the next 14 days and review progress.”
Note: Do not reveal hidden truth explicitly; keep it framed as “explore behaviour patterns”.
